Functions: (Methods / Procedures)

Functions are a way to group code together. Calling the function will run your grouped code.

How to create a function

There are multiple ways to create a function.

Note:

All code must be be between the { } Anything outside of those two brackets is NOT part of the function. 


    function healPlayer(){
        // write all the code that would heal the player here
    }

    // whenever you want to heal the player just call the function
    healPlayer();

    // if you want to heal them twice just call it twice.
    healPlayer();
    healPlayer();

Passing Variables (Parameters, Attributes)

You can pass one or more variables into a function 


    function healPlayer(bodyPart, amount){
        // write all the code that would heal the player here

        // bodyPart - will only heal that body part
        console.log(bodyPart); // Leg

        // amount - will only heal by that amount
        console.log(amount); // 10
    }

    // just heal the leg 10%
    healPlayer('Leg',10);

Return a result

You can a variable back to whatever called this function 


    function getPlayerName(){
        return 'Brad';
    }

    let name = getPlayerName();
    console.log(name); // Brad

Putting it all together with an add function. 


  function add(a, b){
      return a + b;
  }

  let answer = add(10,10);
  console.log(answer); // 20

Return JSON

If you want to return more then 1 thing then create a Object and return that 


  function getWeather(city){
      // write some code that gets the weather then return it
      return {
          rainToday:true,
          sevenDayTemperature:[12,15,8,19,21,15,18,18]
      }
  }

  let weather = getWeather('Toronto');
  // Todays high will be 12
  console.log('Todays high will be '+weather.sevenDayTemperature[0]);

Scope

Any variables created inside your function will not work outside your function 


  function getName(){
      let name = 'Brad';
  }

  console.log(name);  // name is undefined

The opposite is true though. Any variables created outside your function will work inside your function. These are called global variables. They are called that because they work everywhere. For this reason, global variables should be used as little possible.


    let name = 'Brad';                
    function getName(){
        console.log(name);  // Brad        
    }

Reusing variable names


  let name = 'Brad';                
  function getName(){
      let name = 'Dana' // name is now Dana inside this function
      console.log(name);  // Dana
  }
  console.log(name);  // Brad

ES6 Functions

There is a new way to create a function. This is meant for less typing and works with scope slightly better.


    healPlayer=(bodyPart, amount)=>{
        // write all the code that would heal the player here
    }
    healPlayer('Leg',10);
